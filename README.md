# 社团选课系统 · 鄂尔多斯市实验中学

一个给中学用的"网上抢社团"小系统。开抢时间一到,学生在手机或电脑上登录,
一人选一个社团,名额满了就自动停——老师不用再收纸条、画正字、怕重复。

> 仓库代号:OMMS-STXK　|　按 Apache License 2.0 开源分发

---

## 这套系统能做什么

- **学生**:到点登录 → 看到所有社团和"还剩几个名额" → 点一下"报名" → 抢到为止;
  抢错了可以退选改选,一人始终只占一个名额。
- **老师 / 管理员**:提前把学生名单和社团名单导进去,设好"几点开抢",
  开抢后能实时看到每个社团报了多少人、谁还没报,一键导出报名总表。

最关键的一点:**就算一个年级四五百人在同一秒一起点"报名",系统也不会把名额发超**
(不会出现"一个 12 人的社团塞进 13 个人")。这正是它和"用问卷 / 群接龙抢课"最大的区别。

---

## 怎么用(给老师 / 管理员)

1. **登录管理后台**:打开网址,在登录页切到"管理员",输入管理员账号密码。
2. **导入学生**:在"学生"页把名单(姓名、班级、学号)粘贴进去,系统会自动给每人
   生成登录账号和初始密码,可一键导出发给学生。
3. **建社团**:在"社团"页填社团名称和每个社团的名额上限。
4. **设开抢时间**:在"设置"页填"几点几分开抢"。时间没到,学生只能看、不能抢。
5. **开抢后**:在"概览"页实时看各社团报名进度、报名率、未报名名单,随时一键导出表格。

## 怎么用(给学生)

1. 用老师发的账号密码登录(在登录页保持"学生登录")。
2. 没到点只能看;到点后,想去的社团点"报名"即可。
3. 抢到后可在"个人信息"里看自己报的社团,也能"退选"换别的。密码忘了找班主任重置。

## 给技术 / 部署人员

后端是一份单文件 Python(只用标准库)+ SQLite;抢课热路径另有一个可选的 Rust 服务,
配合 Redis 做原子抢占、nginx 做边缘限流。**不接 Redis 也能单机直接跑起来看效果**:

```bash
python3 main.py            # 启动,默认 http://127.0.0.1:2001
```

完整双服务(nginx + Python + 可选 Rust 热服务)一键编排见 `run.sh`;
依赖、架构与各组件说明见 `CONTRIBUTING.md` 与各源码内的中文注释。
运行产生的 `club_system.db` 等数据文件**不会进入仓库**(已在 `.gitignore` 内)。

### 部署安全红线(正式给学生用之前,务必照做)

- **必须启用 HTTPS。** 仓库自带的 `nginx.conf` 默认是开发用的明文 `8080`;正式上线必须改成
  `443` + TLS 证书(`nginx.conf` 末尾留了 443 范例)。否则登录密码、学生账号都是明文过网。
- **管理员初始密码在首次启动时随机生成,并打印在运行窗口里**(形如 `初始密码 xxxxxxxx`)。
  请抄下、立即登录、马上改密。仓库里**不保留任何默认弱口令**。
- **学生初始密码由系统随机生成**,导入名单后一键导出发给学生,请走 HTTPS 下发。
- **校园限流**:若全校共用一个公网出口 IP,边缘限流(默认 30r/s)可能仍偏紧,
  请按规模调高,或对校园出口 IP 段做白名单。真正防超卖的是应用层原子抢占,不是边缘限流。
- **关于"不超卖"**:并发抢占由 Redis 原子操作保证不超卖;若应用进程在"扣名额"与"落库"之间
  被强制杀死,极端情况下可能**少卖**一个名额,重启时会自动对账(rebuild)修正——这对开抢
  那几分钟、且管理员在场的中学场景完全可接受。
